# How I work — the structure (buildable skeleton)

The third layer. `how-i-work-raw.md` = Derek's words (source of truth). `how-i-work-notes.md`
= the quarry (sharpened material). **This file = the shape to build**, so writing + filming has
a frame to pour into. Mindset cut, for derekthomaswood (IC/personal). Agency version forks later.

## The one structural decision that makes it *yours*

Derek's whole thesis is "there's no formal order — reorderable moves, not steps." So a numbered
1→7 pipeline would **betray the point.** The structure has to *show* that it's a palette, not a
sequence. Solution: present the moves as a **set you draw from**, and let the 4 projects show the
same moves recombining in **different orders**. The reorderability IS the structure. That's the
thing a generic "research → wireframes → handoff" page can't do.

## The shape — four parts

**0 — The frame** (short text, top of page). Sets the genre so nobody's hunting for a diamond
diagram. Anchored on the keeper line: *"I design paths to a goal, not screens."* ~2 sentences.

**1 — The spine** (short text, maybe one anchor video). The one thing stated plainly:
- "I don't have a repeatable process. I have a set of reads, and the read decides the move."
- **What I'm reading** (the variables that change the move): how much is already known · product
  problem or team problem · where the risk lives · what fidelity the moment needs · budget & timeline.

**2 — The core: threads + moves** (each gets a short video + a line of text). Explicitly *not*
numbered as steps. Two kinds of thing, and separating them is what keeps this from being a pipeline:

**THREADS — held continuously, set early and re-set as you go** (not steps you pass):

| Thread (Derek's language) | What the video covers | Video |
|---|---|---|
| Goals you can measure against | Set EARLY (in with the listening), then **redefined many times as you go** — collectively adjusted; defining what we DON'T want to be; receipts by design. (Matches GoFundMe: goals + principles in week one.) | ✅ |
| Hardest parts first | Corner the unknowns early so everyone builds on solid ground | ✅ (or woven) |
| Fund exploration | Give people space or they just copy the competition | ✅ (or woven) |
| Accessibility & ethics from the core | Baked in from the very beginning, not a late polish pass | ✅ (or woven) |
| Protect the boundary | Don't collapse explore + finish; `< >` must not become an `X` | ✅ (or woven) |

**MOVES — deployed when the read calls for it** (have a position, even if it shifts):

| Move (Derek's language) | What the video covers | Video |
|---|---|---|
| Listen first | Ask a lot of questions; read systems fast, people slow; the decade-vs-just-started tenure read | ✅ |
| Earn the trust that buys divergence | Can't help them choose from a fake menu; trust is what lets them invest in widening | ✅ |
| Widen the field | Reach ~5 experts + users (they give different leads); screenshots into groups; test what exists | ✅ |
| Make the *right* thing | Paper test / cheapest fidelity that teaches; "make sure it's the right thing to make"; engine before polish, graphics come when it's time | ✅ |
| Build on something real | Low-fi but *real* foundation; everyone uses the real thing; no big reveal | ✅ |

**3 — The proof** (the 4 projects). Same DNA as the existing milestone cards — poster + short
overview + read-more + slides/video. Each project narrated through whichever moves it actually
used, *in the order it actually happened* (different per project — that's the proof). Each carries
its receipt.

## The master plan — the moves × projects grid (fill after picking the 4)

This grid IS the shooting plan and the receipts tracker in one. Rows = moves. Columns = the 4
projects. A filled cell = "this project shows this move — here's the moment/artifact." You don't
film a move in the abstract; you film it *through* whichever project shows it best.

project column has a receipt on the bottom row. Gaps in the grid tell you what's missing before
you shoot a single frame.
